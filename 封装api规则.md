# 核心规范与要求

## 1. 依赖与环境

- **框架**：FastAPI + Uvicorn

- **依赖导入**：读取项目文件分析所需依赖包，通过相对路径正确导入所需模块。

- **启动方式**：`python app.py`（在 `app/` 目录下执行）

## 2. 输入与输出处理

- **数据来源**：同时支持 URL 链接输入和本地路径输入两种方式。

- **参数模型**：强制使用 Pydantic `BaseModel` 定义所有请求体参数；前端统一通过 JSON body 传参。

- **格式对齐（重点）**：所有输入字段名称、类型和输出字段，必须与 `docs/` 目录下 md 文件中的定义严格一致，不得自行增减字段。

## 3. 接口方案选择（二选一）

根据以下规则判断任务类型，选择对应方案，**只实现一种，另一种不得出现在代码中**：

**判断规则**：若 md 文件中满足以下任意一条，则选择情况二；否则选择情况一：

- 明确提到处理时间较长（如 >5 秒、分钟级）

- 涉及 GPU 推理、模型加载、视频/音频处理

- 描述为批量任务、异步处理、队列机制

### 情况一：同步任务

定义 1 个接口：

- `POST /v1/inference`：接收 JSON 请求体，直接返回处理结果。

---

### 情况二：异步/耗时任务

定义 3 个接口：

1. `POST /v1/run_task`：提交任务参数，立即返回 `task_id` 和初始状态。

2. `GET /v1/tasks/{task_id}`：查询当前任务进度或状态。

3. `GET /v1/tasks/{task_id}/download`：
- 将输出文件打包为 `.zip`，存放于 `app/outputs/` 目录。

- 通过 FastAPI `StaticFiles` 挂载 `app/outputs/`，返回格式为：

`{"download_url": "http://{host}:{port}/static/{task_id}.zip"}`
